## 1주차

### LangChain

LangChain은 LLM(대규모 언어 모델)을 활용해 질문응답, 문서 검색, 요약, 챗봇 같은 애플리케이션을 더 쉽게 만들 수 있도록 도와주는 프레임워크

프롬프트, 모델, 메모리, 외부 도구를 연결해 복잡한 작업 흐름을 구성할 때 자주 사용된다.

```python
from openai import OpenAI

client = OpenAI()
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "LangChain을 한 문장으로 설명해주세요"}],
)
text = response.choices[0].message.content

```

- LangChain 이전에는 위와 같은 함수 한번으로 LLM을 호출하는 방식이였다.
단순하고 직관적이지만 실제로는 한 덩어리 안에 여러 역할이 섞여 있다.

1. 프롬프트를 만든다.
2. 모델을 호출한다.
3. 응답 객체에서 원하는 형태를 꺼낸다.
4. 다음 실험을 위해 일부 설정을 바꾸거나 재사용한다.

LangChain을 활용하면 여러 부품들이 연결된 하나의 파이프라인을 통해 LLM의 응답을 얻을 수 있다.

*"정확히 말하면 호출을 조립가능한 객체들로 분해해준다"* 가 더 가깝지 않을까?

```
prompt | llm | parser
```

- prompt : 입력값을 받아 모델이 읽을 메세지 구조로 바꾸는 단계
- llm : 실제 모델 호출 단계
- parser : 모델 출력에서 내가 원하는 결과 모양만 뽑아내는 단계

```python
prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "당신은 {role} 전문가입니다. 핵심만 3줄 이하로 답변하세요."),
        ("human", "{question}"),
    ]
)

prompt = invoke({
  "role" : "파이썬",
  "question" : "리스트 컴프리헨션에 대해서 설명해주세요"
})
```

이와 같이 고정 규칙과 바뀌는 입력을 분리하고, 같은 프롬프트 구조를 여러번 재사용하며 관리할 수 있게 된다.

- parser를 통해 ..
StrOutputParser()는 AIMessage -> str
JsonOutputParser()는 AIMessage -> dict/list

처럼 바꿔주어 사람에게 바로 보여줄 거면 문자열로, 프로그램이 다음 단계에서 처리할 거면 JSON으로 변환해주는 parser를 적절히 선택해 사용하면 된다.

```
입력 -> 프롬프트 템플릿 -> 모델 -> 출력 파서 -> 다음 코드가 쓰기 좋은 값
```

실제 웹에서 사용하는 GPT, Gemini처럼 실시간으로 응답을 작성해나가는 방식은 stream 메소드를 사용한다

```python
for chunk in llm.stream("머신러닝의 3가지 유형을 나열해줘"):
    print(chunk.content, end="", flush=True)
```

만약 여러개의 입력을 동시에 처리하고 싶다면 batch 메소드를 사용하면 된다

```python
questions = ["파이썬 장점 하나 말해줘", "자바스크립트 장점 하나 말해줘", "러스트 장점 하나 말해줘"]
results =llm.batch(questions)
```

1. 사이클 3 : FAQ 검색 함수 만들기

- 질문 문자열을 받아 키워드 매칭으로 관련 FAQ 찾는 함수 만들기

```python
def search_faq(query, faq_data, top_k = 3):
    query = query.strip()
    scored = []

    for faq in faq_data:
        keywords = faq.get("keywords", [])

        score = sum(1 for kw in keywords if kw in query)
        scored.append((score,faq))

    def score_of(item):
        return item[0]

    scored.sort(key=score_of, reverse=True)
    return [faq for _, faq in scored[:top_k]]
```

```python
def build_context(question, faq_data, top_k = 3):
    ref_faqs = search_faq(question, faq_data, top_k= top_k)
    context_parts = []

    for faq in ref_faqs :
        context_parts.append(f"### {faq['category']} - {faq['question']}\n{faq['answer']}\n\n")
    context = "\n\n".join(context_parts)

    return context, ref_faqs

def ask_faq(question, faq_data, client):

      ("human", "{question}")
    ])

    chain = prompt | client | StrOutputParser()
    response = chain.invoke({
      "context":context,
      "question":question
    })

    return response, ref_faqs
```

```python
def rag_chain(question, faq_data, faq_chain_ top_k=3):

    })

      "answer" : answer,

    }
```

```
-> FAQ에서 관련 항목 검색
-> 검색 결과를 context 문자열로 정리
-> 그 context를 LLM에 넣어서 답변 생성
```

1. build_context() 메소드 내부에서 search_faq 를 통해 관련 FAQ를 찾은 후 LLM이 읽기 쉬운 문장으로 합친다.
2. answer 변수에는 실제 그 합쳐진 관련 FAQ의 내용들을 context로 넣고 사용자의 질문 기반으로 답변을 생성한다.
