# 인프라 스터디 1주차 네트워크 기초 복습 및 회고

## 1주차 네트워크 기초

이번 글은 로드밸런싱, NGINX, Kubernetes 같은 상위 개념으로 올라가기 전에  
요청이 실제로 어떤 경로를 따라 서버까지 도달하는지부터 정리해보려는 목적에서 시작했다.

앞선 정리에서 프로세스와 포트 개념을 다뤘다면,  
이번에는 DNS, IP, Port, HTTP, 대역폭, 지연시간이 하나의 흐름에서 어떻게 연결되는지에 집중했다.

이번 주의 핵심 흐름을 한 줄로 쓰면 아래와 같다.

```text
도메인 입력
-> DNS 조회로 IP 확인
-> IP:Port 연결
-> HTTP/HTTPS 요청 처리
-> 네트워크 조건(지연시간/대역폭)에 따라 체감 속도 변화
```

---

### 요청이 서버에 도달하는 과정

![네트워크 전체 흐름 예시](/image/network_flow.png)

> 1. 도메인은 시작점일 뿐이다

브라우저에서 `inho.com`을 입력해도 컴퓨터는 문자열 자체와 통신하지 않는다.  
먼저 DNS 질의를 통해 도메인을 실제 IP 주소로 바꾼 뒤, 해당 IP의 특정 포트로 연결한다.

이후에야 HTTP 요청과 응답이 오가며, 사용자는 비로소 페이지를 받는다.

> 2. 네트워크 기초를 모르면 상위 개념이 흐려진다

프록시나 로드밸런서도 결국은  
"어떤 요청을 어느 IP/Port로 보낼지 결정하는 계층" 위에서 동작한다.

그래서 네트워크의 기본 경로를 이해하지 못하면,  
상위 인프라 기술을 배워도 왜 필요한지 체감하기 어려웠다.

---

### IP, DNS, Port, 그리고 HTTP/TCP

> 1. 세 개념의 역할 구분

- IP: 네트워크 상 장치를 식별하는 주소
- DNS: 도메인 이름을 IP로 변환하는 시스템
- Port: 같은 IP 안에서 어떤 서비스와 통신할지 구분하는 번호

즉 사람이 보는 건 도메인이지만,  
실제 통신은 결국 `IP + Port` 기준으로 이루어진다.

> 2. HTTP와 HTTPS는 전송 계층 위에서 움직인다

HTTP는 요청/응답 규약이고, 일반적으로 TCP 위에서 동작한다.  
HTTPS는 HTTP에 TLS 보안을 더한 형태다.

실무에서는 앞단(NGINX 등)에서 HTTPS를 종료하고,  
내부 구간은 HTTP로 전달하는 구조를 자주 보게 된다.

---

### 대역폭, 지연시간, 처리량

> 1. 느림의 원인을 분리해서 봐야 한다

세 개념은 비슷해 보여도 원인이 다르다.

- 대역폭: 한 번에 전송 가능한 양
- 지연시간: 요청 왕복에 걸리는 시간
- 처리량: 실제로 일정 시간 동안 처리된 데이터 양

비유하면 아래와 같다.

- 대역폭 = 도로 폭
- 지연시간 = 출발지에서 도착지까지 걸리는 시간
- 처리량 = 실제 통행량

요청이 느릴 때는 "회선이 좁다"로 단정하기보다,  
리전 거리로 인한 지연인지 대역폭 한계인지 구분해야 한다.

이때 `ping`, `traceroute`, `curl -w` 같은 도구가 원인 파악에 유용하다.

---

### CDN 캐시 히트/미스 사례

> 1. 리전이 멀면 지연시간이 체감된다

Hugging Face에서 특정 데이터셋을 받을 때 평소보다 느렸던 경험을 복기해보면,  
가까운 리전에 캐시가 없고 먼 리전(예: 영국)에서 처음 가져오는 상황이었다.

이 경우 대역폭이 충분해도 왕복 지연시간이 커서 체감 속도가 크게 떨어진다.

> 2. 유튜브 사례로 보는 캐시 관점

인기 영상은 여러 리전에 미리 복제되어 빠르게 응답된다(캐시 히트).  
반대로 요청이 적은 영상은 가까운 캐시에 없어서 오리진까지 가야 하므로 느릴 수 있다(캐시 미스).

결국 "느리다"는 현상은 회선 문제 하나가 아니라,  
캐시 위치, 리전 거리, 지연시간이 함께 만든 결과라는 점이 중요했다.

## 1주차 네트워크 기초 회고

이번 정리를 통해 가장 크게 얻은 건  
"도구 중심 시선"에서 "요청 흐름 중심 시선"으로 관점을 바꾼 점이었다.

특히 아래 연결을 기준으로 보면 상위 인프라 개념도 훨씬 덜 헷갈린다.

```text
도메인
-> DNS
-> IP/Port
-> HTTP/HTTPS
-> 지연시간/대역폭/캐시 영향
```
