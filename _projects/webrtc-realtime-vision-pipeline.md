## 배경 및 문제 상황

기존 RTSP 입력 스트림 파이프라인은 입력 해상도가 항상 일정하다는 전제로 설계되어 있었습니다.

하지만 실제 운영 환경인 WebRTC의 적응형 비트레이트(ABR) 조건에서는 네트워크 대역폭에 따라 해상도가 실시간으로 변동했습니다.

이로 인해 FFmpeg swscaler 에러, 프레임 버퍼 꼬임, 화면 깨짐, 누적 지연이 발생했고 실시간 관제 시스템의 가용성이 떨어졌습니다.

## 해결 과제 및 수행 내용

### 1. 런타임 해상도 감지 및 데이터 정규화

가변 입력 프레임을 OpenCV Letterbox 알고리즘으로 1280x720 규격에 맞게 정규화했습니다. 이를 통해 후속 AI 추론 모델과 송출 레이어에 전달되는 데이터 정합성을 확보했습니다.

### 2. 제어 계층 분리 설계

FastAPI 기반 제어 API를 구성해 스트림 상태를 실시간 모니터링하고, 해상도 변경 같은 주요 이벤트가 발생해도 프로세스를 전부 중단하지 않고 안전하게 설정을 갱신하거나 재연결할 수 있도록 구조를 개선했습니다.

### 3. 미디어 엔진 튜닝 및 지연 억제

FFmpeg의 `zerolatency` 옵션과 입력 FPS 변화에 연동되는 동적 GOP 설정을 적용했습니다. 이를 통해 타임스탬프 동기화 문제를 줄이고 누적 지연 시간을 기존 대비 30% 이상 개선했습니다.

## 성과

가변적인 네트워크 환경에서도 24시간 중단 없는 실시간 비전 시스템의 안정성을 확보했습니다.

이 경험을 통해 단순 기능 구현을 넘어, 변동성 높은 환경에서의 예외 처리 전략과 유연한 백엔드 아키텍처 설계 역량을 키웠습니다.
